Quick interview with someone who is knowledgeable about technical aspects of CRO.Welcome Rudger de Groot.Conversation about flickering and flashing of A/B tests. Why does it happen and what are the consequences?
Most of the time it is a technical implementation that went wrong.
It could also be settings in your testing tool.
But the problem is that the changes on the page can be noticed by the test population. Test is not objective anymore. Someone in the company might shut down the tests and the test could be abandoned. What should a company look out for when they are implementing the test tool?
Most of the tools are based on JavaScript.
Easy solution: tag manager. But for performance that is not the best thing, makes it asynchronous. You are never sure when it is executed. Recommendation is to put it in head tag. Should it be before any other tags?
Tools don't give very clear advice usually. Should be in the head but after jQuery library. Audience question or comments?What other things can companies do?
Testing tools get test script from CDN. Some tools have single file, but have multiple test experiments. Makes the file unnecessarily big. Archive or remove unused tests from file. What do you recommend about WYSIWYG?
Main benefit to promote the software. I would not recommend it for anything other than minor stuff. Lots of repetition in WYSIWYG files. Should I use synchronous or asynchronous script from my provider?
Sync means you have control over when script is fired. Waiting for document ready event. Can lead to localized or page wide flicker. What do you recommend?
Never wait for DOM ready. It has some risks - don't know if an element is on the page.
Some providers like Optimizely can help with this. Testing on mobile: any tips?
Developers favor jQuery. Disable multiple loads of jQuery if it happens.
Go straight to JavaScript, don't use jQuery. Provides big performance boost.
If you are going to use jQuery - lookup element once and create a variable for it. Sync vs async. Can search engines penalize you for using synchronous script?
Not investigated. Google indexes jQuery and JavaScript. What do you make of that?
I have not tested that either. Any final thoughts?
